### 問題分析

**為什麼規則後處理效果有限？**

原始 OCR 錯誤太嚴重，例如：

```
Ground Truth: 土地登記第三類謄本（所有權個人全部）
OCR 結果:     土:登登記第三類有謄本 (所有權個人人金義5》  全
```

規則後處理只能修正：
- ✅ 已知錯別字（如 "膽本" → "謄本"）
- ✅ 格式問題（如日期、地號）

但無法修正：
- ❌ 上下文相關錯誤（如 "人金義5" 應該是 "全部"）
- ❌ 複雜的文字混淆（如 "土:" 應該是 "土地"）
- ❌ 排版導致的錯誤（如 "班息" 應該是 "正段"）

## 🚀 LLM 後處理方案

### 方案架構

```
原始 OCR (54%)
    ↓
規則後處理 (修正已知錯別字)
    ↓
LLM 智能修正 (理解上下文)
    ↓
預期結果 (75-85%)
```

### 智能策略（根據信心度）

```python
if ocr_confidence < 0.6:
    # 低信心度：全文修正
    strategy = "full"  # 使用 LLM 理解整份文件

elif ocr_confidence < 0.8:
    # 中信心度：欄位修正
    strategy = "fields"  # 只修正關鍵欄位（地號、日期等）

else:
    # 高信心度：不使用 LLM
    strategy = "none"  # 規則修正已足夠
```

```python
# 建立後處理器（啟用 LLM）
postprocessor = TranscriptPostprocessor(
    enable_typo_fix=True,           # 規則修正
    enable_format_correction=True,   # 格式校正
    enable_llm=True,                # 啟用 LLM
    llm_provider="anthropic",       # 使用 Claude
    llm_strategy="auto"             # 智能策略
)

# 執行後處理
corrected_text, stats = await postprocessor.postprocess(
    text=ocr_raw_text,
    ocr_confidence=0.54  # OCR 信心度
)

# 查看結果
print(f"LLM 使用: {stats['llm_used']}")
print(f"估計成本: ${stats['llm_cost']:.6f}")
print(f"錯別字修正: {stats['typo_fixes']} 次")
```

### LLM Prompt 範例

**全文修正 Prompt**：

```
你是專業的台灣地政謄本 OCR 錯誤修正助手。

以下是 OCR 辨識結果，包含許多錯誤。請修正為正確的繁體中文。

OCR 原文：
土:登登記第三類有謄本 (所有權個人人金義5》  全
中焉區中班息三小旋 o221-oooolta中的0
...

修正規則：
1. 保持原有格式和換行
2. 修正常見 OCR 錯誤：
   - 十 → 土, 膽 → 謄, 攝 → 登
   - o/O → 0, l/I → 1
3. 地號格式：XXXX-XXXX
4. 日期格式：民國XXX年XX月XX日
5. 地政術語要正確：所有權、登記、地號、面積

請直接輸出修正後的完整文字，不要添加任何解釋。
```

```
地號標準格式: XXXX-XXXX 或 XXXX-XXXXXXXX
常見 OCR 錯誤:
- o/O → 0
- l/I → 1
- 移除非數字字符

請直接輸出修正後的地號，格式為 XXXX-XXXX，不要解釋。
```

### 可修正的錯誤類型

✅ **LLM 可以修正**：
- 上下文錯誤："人金義5" → "全部"
- 複雜混淆："土:" → "土地", "登登記" → "登記"
- 區段名稱："班息" → "正段"
- 地號格式："o221-oooolta" → "0221-0000"
- 專業術語："所有傑" → "所有權"

❌ **LLM 無法修正**：
- OCR 完全遺漏的文字
- 圖片太模糊導致的錯誤
- 特殊符號完全錯誤

## ❓ 常見問題

### Q1: 為什麼不直接用 LLM 取代 OCR？

A: LLM 需要文字輸入，無法直接讀圖片。必須先用 OCR 提取文字，再用 LLM 修正錯誤。

### Q2: 成本會不會太高？

A: 不會。使用智能策略：
- 高信心度文件（>80%）：不使用 LLM，省成本
- 只有低/中信心度才用 LLM
- 平均成本 ~$0.001/頁，月處理 1000 頁只需 $1

### Q3: 可以用免費的 LLM 嗎？

A: 可以，但不建議：
- 開源 LLM（Llama, Mistral）繁體中文能力較弱
- 需要自己架設伺服器，成本更高
- 建議使用 Claude Haiku（便宜且效果好）

### Q4: 會不會修正錯誤？（把對的改成錯的）

A: 有可能，但機率很低（<5%）：
- LLM 理解台灣地政術語
- Prompt 有明確規則
- 可以人工審查修正結果

### Q5: 處理速度如何？

A:
- 規則後處理：<0.1 秒/頁
- LLM 全文修正：2-5 秒/頁
- LLM 欄位修正：1-2 秒/頁
- 可以 batch 處理提升速度
