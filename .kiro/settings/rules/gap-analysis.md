# 實施差距分析框架

## 目的
識別需求與現有程式碼之間的差距,評估實施策略,並為技術設計提供依據。

## 分析維度

### 1. 現有能力評估
- 現有相關功能的實作方式
- 可重用的元件與模式
- 技術棧與依賴關係
- 程式碼品質與架構

### 2. 差距識別
- 缺失功能清單
- 需要增強的現有功能
- 技術債務與限制
- 相容性考量

### 3. 實施策略評估
- 擴展現有元件 vs 建立新元件
- 重構需求
- 技術風險
- 開發工作量估算

### 4. 整合點分析
- API 介面相容性
- 資料模型擴展需求
- 服務依賴關係
- 配置與部署考量

## 輸出格式

### 差距分析報告結構

1. **執行摘要**
   - 分析範圍
   - 主要發現(3-5 點)
   - 建議策略

2. **現有能力分析**
   - 相關元件清單
   - 功能對照表(需求 vs 現有)
   - 可重用資產

3. **差距詳細分析**
   - 按需求分組的差距清單
   - 技術挑戰
   - 風險評估

4. **實施方案建議**
   - 方案 A: [描述]
     - 優點/缺點
     - 工作量估算
     - 風險等級
   - 方案 B: [描述]
     - 優點/缺點
     - 工作量估算
     - 風險等級
   - 推薦方案與理由

5. **技術研究需求**
   - 需要進一步研究的技術點
   - 外部依賴調查
   - PoC 建議

6. **整合策略**
   - API 設計考量
   - 資料模型變更
   - 向後相容性策略
   - 部署與配置
